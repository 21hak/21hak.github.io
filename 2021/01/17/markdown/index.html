MarkDown 기본 사용법

맠다를 배워야 블로그를 관리하든 뭘 하든 하지.

Posted by wonakk's Blog on Sunday, January 17, 2021

블로그를 만들어 놓고 관리해야지 관리해야지 하다가 일주일이 지난 지금에서야 시작했다.
첫 포스트로 마크다운 사용법에 대해 스스로 정리하면서 글을 작성해볼까 한다.

Reference

https://www.markdownguide.org/basic-syntax/

Markdown이란?

마크다운(markdown)은 일반 텍스트 기반의 경량 마크업 언어다. 일반 텍스트로 서식이 있는 문서를 작성하는 데 사용되며, 일반 마크업 언어에 비해 문법이 쉽고 간단한 것이 특징이다. HTML과 리치 텍스트(RTF) 등 서식 문서로 쉽게 변환되기 때문에 응용 소프트웨어와 함께 배포되는 README 파일이나 온라인 게시물 등에 많이 사용된다.

위키백과에 나오는 정의다. README를 작성할 때 일반적으로 사용되고 지금과 같이 온라인 문서를 편집할 때도 사용된다고 한다. 일반 텍스트에 h1, h2, quote, bold 등등과 같은 효과를 주는데 사용되는 마크업언어로 이해했다.

자주 사용될 것 같은 문법(Syntax)를 위주로 정리해보자.

제목(Header)

문서를 작성할 때 제목은 꼭 필요한 녀석이다. 문서의 구조를 대략적으로 보여주고 어떤 내용인지 감을 수 있도록 한다. <h1>에서 <h6>까지로 표현이 가능하며 1에서 6으로 갈 수록 제목의 크기가 작아진다.

# \<h1> 헤더1

## \<h2> 헤더2

### \<h3> 헤더3

#### \<h4> 헤더4

##### \<h5> 헤더5

###### \<h6> 헤더6

<h1> 헤더1

<h2> 헤더2

<h3> 헤더3

<h4> 헤더4

<h5> 헤더5
<h6> 헤더6

<h1>과 <h2>는 아래와 같이 사용할 수 있다. =와 -는 1개 이상만 있어도 <h1>과 <h2>를 표현할 수 있다.

Header 1 표현
=====

Header 2 표현
----

문단 바꿈, 줄 바꿈

문서를 작성하다 보면 문단을 새로 생성해야하는 경우가 있다. 일반적으로 엔터를 한번 누르면 줄 바꿈이 되지만 마크다운에서는 엔터를 2번 입력하면 문단이 바뀐다. 그러면 새로운 <p> 태그를 여는 효과와 동일하다.

줄바꿈은 스페이스바를 2번 입력하면 된다. 문단 바꿈이나 줄 바꿈이나 비슷하지만 문단 바꿈은 <p>태그를 열고 닫은 것이고 줄바꿈은 <br>태그를 입력한 것이다.

강조(Emphasis)

1. Bold

Bold는 글자를 굵게 처리해서 강조하는 효과를 준다. **나 __로 강조하고자 하는 위치를 감싸주면 된다.

**강조강조** or __강조강조__
강조강조 or 강조강조

2. Italic

글자가 비스듬히 누워있는 Italic로 표현하고 싶은 경우 *문자나 _문자로 원하는 위치를 감싸주면 된다.
*비스듬히* or _비스듬히_
비스듬히 or 비스듬히

3. Bold + Italic

Bold와 Italic을 동시에 표현하고자 하는 경우에는 Bold를 위한 ** 또는 __를 Italic을 위한 * 또는 _와 합쳐주면 된다. 쉽게 말해 *** 또는 ___로 감싸주면 된다. **_와 같이 섞어서 사용도 가능하다.
***Bold and Italic*** or ___Bold and Italic___ or _**Mixed**_
Bold and Italic or Bold and Italic or Mixed

인용문(BlockQuotes)

자신이 인용한 정보와 그 출처를 나타내야하는 경우가 있다. >로 시작하면 해당 문장은 인용으로 표시가 된다.

> 이것은 인용이야!

이것은 인용이야!

여러 문장을 인용으로 표기호가 싶으면 빈 문장도 >로 시작하게 하면 된디ㅏ

> 이것은 이용이야!
>
> 이것도 인용이야!

이것은 이용이야!

이것도 인용이야!

인용문 안에서 마크다운을 사용할 수 있다. 단 모든 syntax를 사용할 수는 없다고 한다. 사용할 수 없는 syntax에 대해서는 정보가 나와있지 않다. 알아서 찾으란다.

> 인용문 안에서
>
> ### 헤더3을 사용해보자

인용문 안에서

헤더3을 사용해보자

> 인용문 안에서
>
> > 인용문을 한번 더 사용해보자

인용문 안에서

인용문을 한번 더 사용해보자

목록(Lists)

1. Ordered Lists

리스트는 평상시에 사용하듯이 숫자. 내용의 형태로 사용하면 된다. 이 때, 숫자는 1,2,3,4와 같은 순서가 아니어도 된다. 1,3,5,2 와 같은 순서로 작성해도 1,2,3,4로 표시된다

1. 첫번째
2. 두번째
3. 세번째
   1. 3-1
   2. 3-2
4. 네번째?
  1. 첫번째
  2. 두번째
  3. 세번째
    1. 3-1
    2. 3-2
  4. 네번째?

Unordered Lists

Unordered lists를 나타낼 때 사용할 수 있는 문자는 *, -, +이다

- 첫번째

* 두번째
  1. 이것은
  2. 마크다운

- 세번째
  • 첫번째
  • 두번째
    1. 이것은
    2. 마크다운
  • 세번째

  ```

  ```
- 리스트 종료!

  • 코드 블럭도 사용해보자

코드(Code)

코드 강조를 하고 싶은 위치를 `로 감싼다.

`는 Backtick(Back Quote)이라고 한다.

html에는 `<h1>` 태그가 있다.

html에는 <h1> 태그가 있다.

code로 강조한 문장 안에서 backtick을 표시하고 싶으면 아래와 같이 사용한다

`이 문장은`으로 감싸져 있습니다.``

code 강조 안에서 `backtick`을 표시하자.

코드 블럭(Code Block)

backtick을 이용한 코드 강조는 단어나 문장 단위에서 주로 사용한다. 좀 더 긴 코드를 표현하고자 할 때는 code block을 사용한다. code block은 4개의 spaces를 입력하거나 </code>로 감싸면 된다. <code>뒤에 html, javascript 등과 같이 해당 code block에서 사용하는 언어를 명시하면 그에 맞게 syntax highlighting이 이뤄진다

    ```python
    if __name__ == "__main__"
        print("hello world")
        print("Let's study markdonw!")
    ```

if __name__ == "__main__"
    print("hello world")
    print("Let's study markdonw!")

    ```html
    <a href="https://21hak.github.io/" target="_blank">Blog</a>
    ```

<a href="https://21hak.github.io/" target="_blank">Blog</a>

    ```javascript
    const a = () => console.log("hello world!")
    ```

const a = () => console.log("hello world!");

개인적으로 backtick을 사용하는게 조금 더 편리한듯 하다.

수평선(Horizontal Rule)

문서를 작성하다 내용의 구분을 명시적으로 표현하고자 할 경우 수평선으로 표시할 수 있다. (수평선이라고 하니 뭔가 바다 저 멀리 수평선을 의미하는것 같다.)
* 또는 - 또는 _를 3번 연속 입력하면 된다. _를 사용할 때, <h2>로 바뀔 수 있기 때문에 엔터를 텍스트를 입력한 뒤 엔터를 한번 입력해준다.

요것도 해보고

---
저것도 해보고
***
다해보자
___

요것도 해보고

저것도 해보고

다해보자

링크(Links)

참고한 블로그의 링크 등을 표시할 떄, url을 그대로 보여주지 않고 원하는 텍스트로 표현해서 문장 사이에 자연스럽게 노출 시킬 수 있다.

내 참된 스승은 바로 이녀석이다. [My Teacher](https://www.google.com)

내 참된 스승은 바로 이녀석이다. My Teacher

Email 링크와 같이 때로는 url을 그대로 보여주고 싶은 경우가 있다. 이럴 경우에는 <로 감싸주면 된다.

<https://21hak.github.io/>
<imsth33@gmail.com>

HTML

마크다운은 문서에 html tag을 이용해 마크다운 syntax를 표현할 수 있다. 이미지를 예를 들면 <img>를 이용해 이미지를 문서에 나타낼 수 있다….는데 왠지 모르게 동작하지 않는다. vscode preview에서는 보이는데 빌드된 후 브라우저에서는 보이지가 않는다. hugo 내부 설정이 있는지 확인해봐야겠다.
